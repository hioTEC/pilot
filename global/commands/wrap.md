整理当前会话的记忆，准备收尾。

## 步骤

### 1. 读取记忆规则和索引
读取 MEMORY.md 获取当前 L0 索引和规则。

### 2. 回顾当前会话，提取候选记忆

逐类检查：

- **decisions**: 本次会话做了哪些非显而易见的决策？（技术选型、架构方向、流程变更等）
- **cases**: 解决了哪些棘手问题？有值得记录的 问题→解法 对吗？
- **patterns**: 发现了哪些可复用的流程或方法？
- **project state**: 项目状态有变化吗？（阶段推进、服务上下线、依赖变更等）
- **references**: 发现了新的外部资源指针吗？

对每个候选，先对照 L0 索引去重。已有的不重复建。

### 3. 写入文件

按类型写入对应位置：
- 决策 → `events/YYYYMMDD-{描述}.md`（不可变）
- 案例 → `cases/case-{问题}-{序号}.md`（不可变）
- 流程 → `patterns/pattern-{名称}.md`（新建或合并已有）
- 项目 → `project/` 或根目录 `project_*.md`（更新已有或新建）
- 指针 → `reference/` 或根目录 `reference_*.md`（更新已有或新建）

每个文件遵循格式：frontmatter + L1（前25行核心信息）+ L2（详情）。

### 4. 更新 MEMORY.md 索引

为每个新建/修改的文件在 Index 对应分类下添加或更新一行 L0 描述。

### 5. 审计摘要

统计本次会话的 harness 合规情况：

```
## /wrap 审计

### 记忆操作
- 读取: 列出本次会话读取了哪些 memory 文件
- 新建: N 个 events, N 个 cases, N 个 patterns
- 更新: N 个 project, N 个 reference
- 跳过: N 个（去重）

### 决策追踪
- 本次做了 N 个非显而易见的决策
- 已记录: N 个
- 未记录（补记或说明原因）: ...

### Cases 咨询
- 本次是否涉及高风险动作？
- 如果是，是否在执行前检索了 cases/？

### 建议
- 下次会话可以从 xxx 继续
- CLAUDE.md 有待更新的内容: ...
```

## 注意事项
- 不确定是否值得记录时，宁可记录
- 会话中如有对 CLAUDE.md 的修改建议但未执行的，在「建议」中提醒
- 审计不是惩罚机制——是帮助迭代 harness 本身的数据源
- $ARGUMENTS 如果非空，作为额外的上下文提示
